# Repository goals

Using [Packer](https://developer.hashicorp.com/packer/install) it generates a [QEMU image](https://www.qemu.org/download/#macos) and [Vagrant Box](http://vagrantup.com) to use the image along side Vagrant.

The Qemu image generated is in raw format because I had specific needs but if you change format to qcow2 it builds much faster and image size is smaller. 

Also had to use accelerator TCG instead of HVF on Mac and that also builds the image slower. If you're on Apple Silicon (M1/M2/M3) building an amd64 Ubuntu image, you can't use hvf. 

The admin password will be changed on first login and there's also a vagrant user which is only allowed to use SSH Key. Both admin and vagrant will have the public key of the human provided in the variables.

# Packer How To

Install the tools from the links above:

```
brew tap hashicorp/tap
brew install hashicorp/tap/packer
brew install qemu
brew install vagrant
vagrant plugin install vagrant-qemu

```

In order to run packer you need to create a SSH key pair for packer to run on the root of this repository:

```
ssh-keygen -t ed25519 -f ubuntu/packer_key -N "" -C packer_key
```

This code also fetches the **public** ssh key from your user and put as the root user of the machine.

If rather have other keys change variables however only absolute paths are allowed or relative to ubuntu folder

You can then run with:

```
packer init ubuntu
packer validate ubuntu
packer build ubuntu
```

*Build Options:*
* -var-file=packer/vars/ubuntu-jammy-server.pkrvars.hcl -> Use a vars file if needed
* -on-error=[cleanup|abort|ask|run-cleanup-provisioner] If the build fails do: clean up (default), abort, ask, or run-cleanup-provisioner.
* -debug -> Press enter on every step

# How to use the image generated by packer

## Truenas

If you want to use raw image on truenas. Create a volume of 32 GB, copy raw file to server and do:

```
dd if=/path/ubuntu-24.04-packer-amd64.raw of=/dev/zvol/<dataset>/<disk_path> bs=1M status=progress
```

## UTM

UTM is the new virtualbox for Mac M series.

Since I am new to UTM. To open with it you have to:
1. Add > Emulate > Other > Skip ISO Boot > Choose machine specs > Save
2. Go to Disk > Delete existing > Add new one > Import > Select image and user Virtio as interface
3. Go to Network > Emulated VLAN with network card virtio-net-pci
4. Hit Save. Go back to Edit > Network > Now you will see Port Forwarding option. Edit the guest port to 22 and host port to anything

## Vagrant

Vagrant it will use the automatic generated file Vagrantfile in the root of the repository but you can change it if you would like.

Run `vagrant up`

It can take some time for machine to boot. If some how you see a SSH port error it could be vagrant hasn't release it. You can find the process and kill it with `lsof -i :50022`

### Running ansible from Vagrantfile
Add the below to your Vagrantfile

```
Vagrant.configure("2") do |config|
  config.vm.provision "ansible_local" do |ansible|
    ansible.playbook = "myansible/playbook.yml"
    ansible.verbose = true
  end
end
```

### Defining names and hostnames in Vagrant
```
Vagrant.configure("2") do |config|
  # Hostname
  config.vm.hostname = "myubuntuvm"
  # Replace default in vagrant
  config.vm.define "myubuntuvm"
  # Name in GUI
  config.vm.provider :virtualbox do |vb|
    vb.name = "myubuntuvm"
  end
end
```

### Define multiple machines in one file

Once you define multi machines in one vagrantfile you can target a machine using vagrant

```
vagrant up myubuntuvn
```

Here is a snipet

```
Vagrant.configure("2") do |config|

  config.vm.define "myubuntuvm" do |myubuntuvm|

    myubuntuvm.vm.box = "ubuntu-20.04-desktop-amd64"
    myubuntuvm.vm.hostname = "myubuntuvm"
    myubuntuvm.vm.provider :virtualbox do |virtualbox|
      virtualbox.name = "myubuntuvm"
    end
  end

  config.vm.define "mydevvm" do |mydevvm|

    mydevvm.vm.box = "ubuntu-19.10-desktop-amd64"
    mydevvm.vm.hostname = "mydevvm"
    mydevvm.vm.provider :virtualbox do |virtualbox|
      virtualbox.name = "mydevvm"
    end
  end
end
```
